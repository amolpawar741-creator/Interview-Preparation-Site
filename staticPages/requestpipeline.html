ASP.NET Core Request Pipeline

1. Client Sends Request

Browser or API client sends an HTTP request (GET, POST, etc.).

2. Kestrel Web Server

Kestrel receives the request and creates an HttpContext object.

3. Middleware Pipeline

Request flows through middleware components.

  • Can process request
  • Can call next middleware
  • Can stop pipeline
4. Routing

Matches request URL to the correct endpoint (Controller/Action).

5. Authentication & Authorization

Validates user identity and permissions.

6. Endpoint Execution

Controller action executes business logic and model binding occurs.

7. Response Creation

Returns JSON, View, or status code as response.

8. Response Through Middleware

Response flows back through middleware (logging, compression, etc.).

9. Response Sent to Client

Kestrel sends the final response back to the client.

Interview Summary:

Request → Kestrel → Middleware Pipeline → Routing → Controller → Response → Middleware → Client
