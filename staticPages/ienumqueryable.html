Concept

  • IEnumerable → Works in memory
  • IQueryable → Works on database
  • List → Fully loaded data

Real DB Example

Assume table has 1,00,000 records

IEnumerable

var data = context.Users.AsEnumerable() .Where(x => x.Age > 30) .ToList();

🔴 First loads ALL 1,00,000 records from DB
🔴 Then filters in memory
❌ Slow & high memory usage

IQueryable

var data = context.Users .Where(x => x.Age > 30) .ToList();

🟢 SQL runs in DB: SELECT * FROM Users WHERE Age > 30
🟢 Only filtered data comes
✅ Fast & efficient

List

var data = context.Users.ToList(); var result = data.Where(x => x.Age > 30);

🔴 DB hit already done
🔴 All data loaded in memory
❌ Filtering happens after loading

Key Difference (DB Impact)

  • IQueryable → Filtering in DB (best performance)
  • IEnumerable → Filtering in memory (bad for large data)
  • List → Data already fetched (no DB optimization)

Interview One-Liner

IQueryable executes query in database, while IEnumerable and List bring data into memory first, which impacts performance for large datasets.
